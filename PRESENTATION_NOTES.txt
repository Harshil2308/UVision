Sunlight Exposure & Vitamin D Recommendation System
Presentation Notes

1. Project Title

Sunlight Exposure & Vitamin D Recommendation System


2. One-Line Project Idea

This project is a full-stack health monitoring system that combines IoT, MySQL, Python AI/ML, and a web dashboard to provide sunlight exposure and Vitamin D recommendations.


3. Problem Statement

Many people do not know:

- how much sunlight they need
- when sunlight is safe
- how UV conditions affect Vitamin D generation

This project helps solve that by combining:

- live UV data
- user profile data
- health data
- recommendation logic
- machine learning prediction


4. Core Objective

The main objective of the project is:

- to collect UV-related data
- to store and manage it in MySQL
- to analyze it using Python logic and ML
- to show useful results on a web dashboard
- to provide personalized Vitamin D recommendations

9. Hardware / IoT Part

Arduino reads sensor values and sends them through serial communication.

Python script:

- reads serial data from COM port
- converts it into UV-related values
- stores it into MySQL

Script used:

- python/iot/uv_serial_reader.py

Modes supported:

- simulation mode
- real serial mode


10. AI / Recommendation Part

The project uses two AI-related approaches:

1. Rule-based recommendation logic

- implemented in Python
- calculates safe duration
- estimates Vitamin D
- gives risk level
- gives best exposure time window

File used:

- python/ai/recommendation_engine.py

2. ML model trained from dataset

- trained using weather-related features
- predicts UV index
- model used: RandomForestRegressor

Training file:

- python/ml/train_model.py

Prediction API file:

- python/ml/app.py


11. Dataset Usage

We also used a dataset for model training.

Dataset purpose:

- to support future intelligent recommendation improvements

Current ML target:

- uv_index

Features used in current ML code:

- temperature_celsius
- humidity
- wind_kph
- pressure_mb
- cloud
- feels_like_celsius
- visibility_km

Model output file:

- uv_model.pkl


12. High-Level System Architecture

System flow:

1. Arduino collects UV sensor reading
2. Python reads sensor data via serial USB
3. Python stores data in MySQL
4. Node.js backend fetches data from MySQL
5. Frontend dashboard displays live values and analytics
6. Python recommendation engine calculates recommendation
7. ML model can predict UV-based patterns using dataset data


13. Current Workflow In The Project

User workflow:

1. User logs in
2. Dashboard loads data from backend
3. Live UV data is shown
4. User can save exposure logs
5. User can update health lab values
6. Admin can monitor system data
7. AI page can calculate recommendation

Hardware workflow:

1. Arduino sends UV data
2. Python receives data
3. MySQL stores data
4. Backend reads latest UV values
5. Dashboard shows live UV updates

ML workflow:

1. Dataset is stored in MySQL
2. Python training script reads training table
3. Model is trained
4. Model is saved as uv_model.pkl
5. Flask API loads model
6. Prediction API returns UV prediction

15. Exact Startup Order For Presentation

Step 1:

Open project folder in PowerShell

Path:

c:\Users\harsh\Desktop\learn code\UVision

Step 2:

Make sure MySQL is running

Step 3:

Start backend

Command:

npm start

Expected:

UVision backend running on http://localhost:4000

Step 4:

Start frontend

Command:

python -m http.server 5500

Expected:

Frontend opens at:

http://localhost:5500

Step 5:

If you want to show ML model API, start Flask app in another terminal

Command:

python python/ml/app.py

Expected:

Flask API starts on:

http://localhost:5000

Step 6:

If you want to show hardware demo, connect Arduino to laptop

Important:

- first confirm backend and frontend are already running
- then connect Arduino
- make sure Serial Monitor is closed before running Python serial reader

Step 7:

Run serial reader if showing hardware

Command:

python python/iot/uv_serial_reader.py --mode serial --port COM3 --baud 9600

If hardware is not being shown, use simulation mode:

python python/iot/uv_serial_reader.py --mode simulate --max-reads 5
